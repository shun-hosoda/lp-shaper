1. 主要な海外競合（競合の技術的欠陥）
A. Framer AI / Wix ADI
現状: UIデザインツールがAIを組み込んだ形。
No, because: 自由度が高すぎて、初心者が触ると逆にデザインが崩れます。また、「マーケティングの型」ではなく「見た目の美しさ」に特化しているため、成果が出るLPになりにくい。
あなたの勝機: 「デザインをさせない」という制約。
B. Durable / 10Web
現状: 「30秒でサイト公開」を謳い、AIが構成から画像選定まで行う。
No, because: 生成されるサイトが「テンプレートの流し込み」に過ぎず、どれも同じ顔をしています。また、独自のCMSにロックインされるため、データのポータビリティが皆無です。
あなたの勝機: 「分析（競合LPの成功パターン反映）」に基づいた、動的なコンテンツ生成。
C. Relume (React + Tailwind 構成の覇者)
現状: AIがサイトマップとワイヤーフレームを作り、FramerやWebflowへエクスポートする。
評価: 開発者向けとしては最強。
No, because: 「パパっと作りたい非エンジニア」にはハードルが高すぎる。
あなたの勝機: エクスポート先（Vercel等）をインフラごと隠蔽し、完全に抽象化すること。

2. 既存サービスが解決できていない「技術的負債」
海外の先行サービスを分析すると、以下の3点が「構造的な弱点」として残っています。ここを突く設計にしてください。
「コピーライティング」の凡庸さ: 多くのAIはWikipediaのような説明文を書きますが、LPに必要な「煽り」や「共感」が書けません。
マルチデバイス最適化（レスポンシブ）の破壊: AIが自由にHTMLを書くと、特定の端末でレイアウトが崩れます。
SEO・速度の軽視: 大量のライブラリを読み込み、Lighthouseのスコアが低い。

3. あなたが勝つための「アーキテクチャ的差別化」
競合を凌駕するために、以下の「インテリジェント・コンポーネント方式」を提案します。
技術スタックの指定
LLMに自由記述させるのではなく、「型定義されたコンポーネント・レジストリ」を構築してください。
TypeScript
// registry/hero-sections.ts
export const HeroVariants = {
  "minimal-direct": {
    component: "HeroA",
    requiredProps: ["title", "ctaText"],
    validation: (data) => data.title.length < 20 // 爆速で読ませるための制約
  },
  "story-driven": {
    component: "HeroB",
    requiredProps: ["painPoint", "solutionText", "bgImage"],
  }
}

実装すべき「キラー機能」
競合URLスクレイピング機能: 競合URLを入れると、その構成（Headline, Social Proof, FAQ）をリバースエンジニアリングして、自社のスキーマにマッピングする機能。これは海外サービスでも「分析」止まりで「生成への直接注入」はまだ弱いです。
