## Phase 1: 初見ラウンド

**PM**: publishLp で既公開プロジェクトの再公開制御なし。KPIイベント重複記録の可能性。MVP範囲として許容確認必要。

**Arch**: `NEXT_PUBLIC_APP_URL` 未設定時に `localhost:3100` がDBに書き込まれるフォールバックが危険。本番での静かなデータ汚染リスク。

**Eng**: `type Db = SupabaseClient<any>` + `as unknown as Db` による型安全性の欠如。`satisfies` パターンで戻り値は担保されており合格ライン。

**Sec**: `lp_events` INSERT時に他人の project_id 偽装リスクを懸念 → RLSポリシー確認で解消（EXISTS句で owner_user_id チェック済み）。

**DB**: `MAX(version_no)+1` の楽観採番。並行リクエストで UNIQUE制約違反の可能性。MVPでは極低リスク。
